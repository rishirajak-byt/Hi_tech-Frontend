1. Executive Summary
This Product Requirements Document (PRD) defines the full scope, functional specifications, and technical requirements for Phase 1 of the HiTech University Management System (UMS). Phase 1 targets two interconnected deliverables: (1) a public-facing College Website and (2) a Student Portal with core academic management features.
The system is designed to meet the operational needs of a typical B.Tech-offering engineering college in India, accommodating multi-semester structures, Indian academic calendars, exam patterns (CIA + End Semester), CGPA grading as per AICTE norms, and role-based access for Administrators and Students.
The implementation will be entirely frontend (React.js or Next.js), with Supabase used exclusively for authentication (login/session management). All data in Phase 1 will be seeded as realistic dummy/mock data — no live database reads or writes are required beyond authentication.

5. Deliverable 2 — Student Portal
The Student Portal is a secure, role-based web application accessible only to enrolled students upon authentication. It serves as the student's central hub for academic information, attendance, results, documents, and institutional notifications.
5.1 Authentication (Supabase)
Auth Aspect
Specification
Auth Provider
Supabase Auth (email + password login); no OAuth in Phase 1
Student Login
Unique college email ID (e.g., 2023cse001@college.edu.in) + password
Session Management
JWT-based session; persistent across browser refresh; auto-logout on token expiry
Password Reset
Email-based password reset via Supabase (enabled by default)
Role Detection
User metadata in Supabase stores role: 'student' or 'admin'; portal loads accordingly
Protected Routes
All portal routes redirect to login page if user is not authenticated
Student Data Binding
Each login email maps to a dummy student profile in a static JSON/JS data file
Admin Login
Separate admin view accessible at /admin; same Supabase auth, different role
5.2 Student Portal Layout & Navigation
Persistent left sidebar navigation (collapsible on mobile)
Top header bar with: College logo, Student name, Notification bell, Profile avatar
Sidebar menu items: Dashboard | Attendance | Results & Marks | Documents | Notifications | Profile
Color scheme: Navy Blue primary with white content area; Indian academic feel
Breadcrumb navigation on all inner pages

5.5.4  Marksheet Upload (Admin Feature)
Admin can upload PDF marksheets for each student from the Admin panel
Upload interface: Select student roll number, select semester, attach PDF file
Uploaded files stored in Supabase Storage bucket (optional — or as static asset in Phase 1)
Student can then download the uploaded marksheet from their Results page

5.6 Notification System
A notification bell icon is persistently visible in the top navigation bar. It alerts students to important updates from the college administration.
5.6.1  Notification Icon Behavior
Bell icon in top navbar with an unread count badge (e.g., red badge showing '3')
Clicking the bell opens a dropdown panel with the 5 most recent notifications
'Mark all as read' button to clear the badge
'View all' link navigates to the full Notifications page

5.7 Documents Module
Students can upload and manage personal and academic documents within the portal. This replaces the traditional physical document submission process.
5.7.1  Document Categories
Identity Documents: Aadhar Card, PAN Card, Passport
Academic: Class 10 Marksheet, Class 12 Marksheet, Migration Certificate, Transfer Certificate
College Issued: Admission Letter, ID Card (downloadable PDF), Bonafide Certificate
Medical: Medical Fitness Certificate, Disability Certificate (if applicable)
Other: Income Certificate, Caste Certificate, Sports Achievement
5.7.2  Document Upload Features
Upload button per document category; supported formats: PDF, JPG, PNG (max 5MB)
Status indicator per document: Not Uploaded | Uploaded | Verified | Rejected
Upload date and last modified timestamp displayed
View/preview uploaded document in modal
Delete and re-upload option (admin approval required for re-upload of key documents)
Download button for college-issued documents
6. Admin Panel — Basic View (Phase 1)
Phase 1 includes a basic read-only Admin Dashboard for the college administrator. It consolidates data views across enrollment, attendance, and fee status. Full admin CRUD functionality is scoped for Phase 2.
6.1 Admin Dashboard Stats
Total enrolled students across all batches and branches
Department-wise student count (CSE, ECE, ME, CE, EE)
Overall attendance statistics: Average attendance % across all students
Students with attendance below 75% — count and list
Fee collection summary: Total collected vs total pending (dummy data)
Recent admin activity log (last 10 actions — dummy entries)
6.2 Admin — Student List View
Searchable and filterable student roster table
Columns: Roll No, Name, Branch, Semester, Attendance %, CGPA, Fee Status
Click on student to view their full profile (read-only in Phase 1)
Filter by: Department, Year, Semester, Fee Status, Attendance Status
6.3 Admin — Marksheet Upload
Upload interface for admin to attach marksheet PDF to a student record
Select student by roll number (searchable dropdown from dummy list)
Select semester for which marksheet is being uploaded
File upload (PDF only, max 5MB)
Confirmation toast on successful upload
Uploaded files will become available in the student's Results page
